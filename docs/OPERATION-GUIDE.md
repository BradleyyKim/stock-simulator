# 모의투자 시뮬레이션 — 운영 가이드

## 1. 현재 구현된 기능

### 학생 화면 (`/student`)

| 기능 | 상태 | 설명 |
|------|------|------|
| 로그인 | ✅ 완료 | 이름 드롭다운 + 휴대폰 뒷자리 4자리 PIN |
| 종목 현황 | ✅ 완료 | 5개 종목의 현재가, 변동률, 등락폭 표시 |
| 캔들스틱 차트 | ✅ 완료 | 종목 클릭 시 라운드별 가격 변동 차트 (Lightweight Charts) |
| 매수/매도 | ✅ 완료 | 수량 입력, 잔여현금/최대수량 자동 계산, 현금/보유량 검증 |
| 포트폴리오 | ✅ 완료 | 총자산, 보유현금, 투자금, 종목별 보유수량/평가액 |
| 순위판 | ✅ 완료 | 전체 학생 자산 순위, 수익률 표시, 본인 하이라이트 |
| 힌트 패널 | ✅ 완료 | 1/2/3단계 힌트가 관리자 공개 시점에 맞춰 순차 표시 |
| 실시간 동기화 | ✅ 완료 | Firebase Realtime DB를 통해 모든 데이터 실시간 반영 |

### 관리자 화면 (`/admin`)

| 기능 | 상태 | 설명 |
|------|------|------|
| 학생 등록 | ✅ 완료 | 텍스트로 이름/PIN 일괄 등록 |
| 시나리오 선택 | ✅ 완료 | 미리 정의된 시나리오 선택 → 종목 초기화 |
| 라운드 제어 | ✅ 완료 | 시작 → 거래 진행 → 마감/결과 반영 → 다음 라운드 준비 |
| 힌트 공개 | ✅ 완료 | 1단계 → 2단계 → 3단계 순차 공개 버튼 |
| 가격 반영 | ✅ 완료 | 라운드 마감 시 시나리오의 변동률로 자동 계산 반영 |
| 학생 현황 | ✅ 완료 | 전체 학생의 현금, 총자산, 수익률, 접속 상태 테이블 |
| 순위판 on/off | ✅ 완료 | 학생 화면의 순위판 공개/비공개 토글 |

### 시뮬레이션 시스템

| 기능 | 상태 | 설명 |
|------|------|------|
| 턴제 시장 | ✅ 완료 | 관리자가 라운드를 넘겨야만 상태 변경 |
| 5개 가상 종목 | ✅ 완료 | 인공지능(10,000), 에너지(8,000), 바이오(12,000), 반도체(15,000), 게임(9,000) |
| 시나리오 기반 운영 | ✅ 완료 | 2개 시나리오 × 6라운드, 라운드별 뉴스/힌트/변동률 사전 정의 |
| 주문 검증 | ✅ 완료 | 현금 부족 매수 차단, 보유량 부족 매도 차단 |
| 자산 자동 계산 | ✅ 완료 | 현금 + (보유수량 × 현재가) 자동 재계산 |

## 2. 게임 상태 흐름 (Phase Lifecycle)

```
waiting → trading → reviewing → waiting → trading → ... → ended
  │          │          │
  │          │          └─ "다음 라운드 준비" 클릭
  │          └─ "거래 마감 및 결과 반영" 클릭
  └─ "라운드 N 시작" 클릭
```

각 상태에서 가능한 동작:

| 상태 | 관리자 가능 동작 | 학생 화면 |
|------|-----------------|----------|
| `waiting` | 라운드 시작 | "대기 중" 표시, 거래 불가 |
| `trading` | 힌트 1/2/3 공개, 거래 마감 | 거래 가능, 힌트 순차 확인 |
| `reviewing` | 다음 라운드 준비 | 결과 확인, 거래 불가 |
| `ended` | - | 최종 결과 |

## 3. 관리자 행동 가이드 (수업 운영 매뉴얼)

### 수업 전 준비 (10분)

```
1. /admin 접속
2. "학생 등록" 클릭
3. 학생 이름과 PIN 입력 (한 줄에 하나씩):
   ┌──────────────────────┐
   │ 홍길동, 1234          │
   │ 김철수, 5678          │
   │ 이영희, 9012          │
   │ ...                   │
   └──────────────────────┘
4. "학생 등록하기" 클릭
5. 시나리오 선택 (예: "시나리오 1: 기본 시장 체험")
   → 자동으로 종목 초기화, 학생 자산 100,000원으로 리셋
6. 학생들에게 접속 URL 공유 (/, 또는 QR코드)
```

### 라운드 진행 (20분/라운드)

```
시간     관리자 행동                       학생 화면 변화
──────────────────────────────────────────────────────────

0분     "라운드 N 시작" 클릭              → 상태가 "거래 가능"으로 변경
        ↓ 힌트가 시나리오에서 자동 로드됨    학생은 아직 힌트 안 보임

4분     "힌트 1단계 공개" 클릭             → 힌트 패널에 1단계 표시
        (예: "정부가 경제 부양을 위한        "정부가 경제 부양을 위한
         정책을 검토 중이다.")               정책을 검토 중이다."

8분     "힌트 2단계 공개" 클릭             → 1단계 + 2단계 표시
        (예: "금융 정책의 변화가              학생들 토의/판단 수정
         예상된다.")

12분    "힌트 3단계 공개" 클릭             → 1+2+3단계 모두 표시
        (예: "대출과 투자에 긍정적인          학생들 최종 판단
         영향을 줄 수 있는 조치다.")

12~16분  학생들이 매수/매도 주문             종목 탭 → 종목 클릭 → 주문

16분    "거래 마감 및 결과 반영" 클릭        → 상태 "결과 확인"
        ↓ 시나리오 변동률 자동 적용           가격 변동, 순위 변경 확인
        ↓ 전체 학생 자산 재계산               (거래 불가)

        뉴스 발표 (관리자 화면에 표시됨):
        "정부, 기준금리 0.25%p 인하 발표"

18~20분  결과 논의 시간 (교사 재량)

20분    "다음 라운드 준비" 클릭             → 상태 "대기 중"
        (다음 라운드 반복)
```

### 시나리오별 라운드 내용 (현재 내장)

**시나리오 1: 기본 시장 체험**

| 라운드 | 최종 뉴스 | 주요 변동 |
|--------|----------|----------|
| 1 | 정부, 기준금리 0.25%p 인하 발표 | AI +5%, 반도체 +4% |
| 2 | AI 반도체 수요 폭발, 글로벌 공급 부족 | AI +12%, 반도체 +8% |
| 3 | 중동 지역 무력 충돌, 원유 공급 불안 | 에너지 +10%, 반도체 -6% |
| 4 | 대형 게임사 대표 사임 및 구조조정 | 게임 -12% |
| 5 | 바이오 기업, 신약 임상 3상 성공 | 바이오 +15% |
| 6 | 정부, 반도체 산업 대규모 지원 발표 | 반도체 +15%, AI +6% |

**시나리오 2: 변동성 장세**

| 라운드 | 최종 뉴스 | 주요 변동 |
|--------|----------|----------|
| 1 | AI 기술 혁명 선언, 투자 확대 | AI +15%, 반도체 +6% |
| 2 | 대형 AI 기업 회계 부정 스캔들 | AI -18%, 반도체 -5% |
| 3 | 글로벌 경기 침체 우려 확산 | 전 종목 하락 (반도체 -10%) |
| 4 | 정부 긴급 경기 부양책 발표 | 전 종목 상승 (반도체 +12%) |
| 5 | 에너지 대란, 원자재 가격 급등 | 에너지 +20%, 반도체 -8% |
| 6 | 블록버스터 게임 출시 대성공 | 게임 +18% |

### 비밀정보 운영 (현재 오프라인 방식)

```
1. 관리자가 사전에 정보 코드를 종이/메모에 준비
   예: A1 = "특정 산업에 정부 정책이 검토 중이다." (1,000원)
       A2 = "첨단 제조업 관련 분야가 수혜 가능성." (3,000원)
       A3 = "고성능 연산 장치 관련 산업이 직접 수혜." (5,000원)

2. 학생이 "선생님, A2 살게요" → 교사가 해당 힌트를 알려줌

3. 교사가 해당 학생의 현금을 수동으로 차감
   (Firebase 콘솔에서 직접 또는 추후 앱 내 기능 추가)
```

### 수업 종료 시

```
1. 최종 라운드 결과 확인
2. 순위판 공개 (이미 on 상태라면 그대로)
3. 1위~3위 발표
4. 결과 토론 / 전략 분석 / 느낀 점 공유
```

## 4. 학생 행동 가이드 & 기대값

### 학생이 할 수 있는 행동

| 행동 | 조건 | 결과 |
|------|------|------|
| 로그인 | 이름 선택 + PIN 4자리 | 학생 화면 진입 |
| 종목 확인 | 언제든 | 5개 종목의 현재가/변동률 확인 |
| 차트 확인 | 종목 클릭 | 라운드별 캔들스틱 차트 |
| 매수 | 거래 가능 상태 + 현금 충분 | 현금 감소, 보유수량 증가 |
| 매도 | 거래 가능 상태 + 보유수량 있음 | 현금 증가, 보유수량 감소 |
| 포트폴리오 확인 | 언제든 | 총자산, 현금, 보유 종목 확인 |
| 순위 확인 | 순위판 공개 상태 | 전체 순위, 본인 순위 확인 |
| 힌트 확인 | 관리자가 공개한 힌트 | 순차적으로 표시됨 |

### 주문 검증 규칙

| 상황 | 시스템 반응 |
|------|-----------|
| 거래 불가 상태에서 주문 | "현재 거래가 불가능한 상태입니다." |
| 현금 < 매수 금액 | "현금이 부족합니다." |
| 보유수량 < 매도 수량 | "보유 수량이 부족합니다." |
| 정상 주문 | "주문이 체결되었습니다." → 0.8초 후 자동 닫힘 |

### 자산 계산 방식

```
총자산 = 보유 현금 + Σ(각 종목 보유수량 × 현재가)

예시:
  현금: 40,000원
  인공지능 3주 × 10,500원 = 31,500원
  반도체 2주 × 16,500원 = 33,000원
  총자산 = 40,000 + 31,500 + 33,000 = 104,500원
  수익률 = (104,500 - 100,000) / 100,000 = +4.5%
```

### 기대되는 학생 행동 패턴

| 상황 | 예상 행동 | 교육 포인트 |
|------|----------|------------|
| 1단계 힌트 공개 | 모호해서 판단 못함 → 관망 | 불완전한 정보에서의 의사결정 |
| 2단계 힌트 공개 | 방향성 파악 → 일부 매수 시작 | 정보 해석 능력 |
| 3단계 힌트 공개 | 확신 → 적극 매수 또는 이미 늦음 | 타이밍의 중요성 |
| 급락 이벤트 발생 | 패닉셀 or 저가 매수 | 공포와 탐욕, 역발상 투자 |
| 특정 종목 몰빵 후 급락 | 큰 손실 → 분산투자 필요성 체감 | 리스크 관리 |
| 타인 순위 확인 후 따라 매수 | 군중심리 경험 | 정보 vs 군중 |

## 5. 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| 학생이 로그인 안됨 | PIN 불일치 또는 미등록 | /admin에서 학생 재등록 |
| 거래가 안됨 | phase가 'trading'이 아님 | 관리자가 라운드 시작했는지 확인 |
| 가격이 안 변함 | 라운드 마감 안 함 | "거래 마감 및 결과 반영" 클릭 |
| 순위판이 안 보임 | leaderboardVisible = false | /admin에서 순위판 공개 토글 |
| 화면이 안 바뀜 | Firebase 연결 문제 | 와이파이 확인, 새로고침 |
| 데이터 초기화하고 싶음 | - | 시나리오 재선택 → 종목/자산 리셋됨 |
