# 기획 대화 아카이브

> 이 문서는 기획자(수학 교사 친구)가 ChatGPT와 나눈 원본 대화의 핵심 요약입니다.
> 전체 원본은 `schoolProject.md`에 있으나, 개발에 필요한 내용은 모두 `PROJECT-SPEC.md`와 `OPERATION-GUIDE.md`에 반영되어 있습니다.

## 대화 흐름 요약

### 1단계: 아이디어 도출
- 고등학교 창체 동아리로 모의투자 경연 구상
- 런닝맨 주식 시뮬레이션 영상 레퍼런스 (https://youtu.be/8IyCPgwrJ24)
- 10분 라운드 × 12라운드 = 2시간 수업 구조

### 2단계: 구글시트 프로토타입 설계
- 구글폼(거래 입력) + 구글시트(자동 계산) 조합 검토
- 시트 구조: 시장, 학생, 거래기록, 포트폴리오, 순위판, 학생조회
- SUMIFS/XLOOKUP 수식으로 자동 계산 설계

### 3단계: 웹앱 필요성 인식
- 학생 휴대폰 직접 입력 요구 → 구글폼의 한계
- 관리자 대시보드 필요 → 시트로는 불편
- 개발자 친구(본인)에게 의뢰 결정

### 4단계: 요구사항 구체화
- 런닝맨식 1/2/3단계 힌트 시스템
- 비밀정보 코드 구매 시스템 (정보의 비대칭성)
- 14명 개인전, 매수/매도 모두 가능, 주 단위 거래
- 개인 포트폴리오 확인 기능

### 5단계: 10차시 커리큘럼 설계
- 1~3차시: 기본 시장 이해
- 4~6차시: 정보 비대칭 + 전략 + 군중심리
- 7~8차시: 미국 시장 + 환율
- 9차시: 공매도/선물 (선택)
- 10차시: ETF 제작 + 최종 리그

### 6단계: 개발 요구사항 문서화
- 턴제 시장, 학생/관리자 화면 분리
- Firebase/Supabase BaaS 추천
- 3단계 개발 우선순위 (MVP → 정보 시스템 → 고급 기능)

## 핵심 결정사항 (대화에서 확정)

| 항목 | 결정 |
|------|------|
| 대상 | 고등학생 ~15명, 개인전 |
| 종목 | 가상 산업 5개 (인공지능, 에너지, 바이오, 반도체, 게임) |
| 시작자산 | 100,000원 |
| 거래방식 | 주(株) 단위 매수/매도 |
| 시장구조 | 턴제 (교사가 라운드 제어) |
| 정보구매 | 1급 1,000원 / 2급 3,000원 / 3급 5,000원 |
| 거래규칙 | 공매도/레버리지 없음, 현금/보유량 검증 |
